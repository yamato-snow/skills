アルゴリズム哲学は、コードを通じて表現される計算美学の運動です。.mdファイル（哲学）、.htmlファイル（インタラクティブビューア）、.jsファイル（ジェネレイティブアルゴリズム）を出力します。

これは2つのステップで行われます：
1. アルゴリズム哲学の作成（.mdファイル）
2. p5.jsジェネレイティブアートの作成による表現（.html + .jsファイル）

まず、このタスクに取り組みます：

## アルゴリズム哲学の作成

まず、以下を通じて解釈されるアルゴリズム哲学（静的画像やテンプレートではない）を作成します：
- 計算プロセス、創発的振る舞い、数学的美
- シード付きランダム性、ノイズフィールド、有機的システム
- パーティクル、フロー、フィールド、力
- パラメトリック変動と制御されたカオス

### 重要な理解
- 受け取るもの：ユーザーからの微妙な入力や指示を考慮に入れるが、基盤として使用する；創造的自由を制約すべきではない
- 作成するもの：アルゴリズム哲学/ジェネレイティブ美学運動
- 次に起こること：同じバージョンが哲学を受け取り、それをコードで表現する - 90%アルゴリズム生成、10%必須パラメータのp5.jsスケッチを作成

このアプローチを検討：
- ジェネレイティブアート運動のマニフェストを書く
- 次のフェーズでそれを生命に吹き込むアルゴリズムを書く

哲学は以下を強調する必要があります：アルゴリズム的表現。創発的振る舞い。計算美。シード付き変動。

### アルゴリズム哲学の生成方法

**運動に名前を付ける**（1-2語）：「Organic Turbulence」/「Quantum Harmonics」/「Emergent Stillness」

**哲学を明確にする**（4-6段落 - 簡潔だが完全）：

アルゴリズムの本質を捉えるために、この哲学がどのように現れるかを表現：
- 計算プロセスと数学的関係？
- ノイズ関数とランダム性パターン？
- パーティクルの振る舞いとフィールドダイナミクス？
- 時間的進化とシステム状態？
- パラメトリック変動と創発的複雑さ？

**重要なガイドライン：**
- **冗長性を避ける**：各アルゴリズム的側面は一度だけ言及。新しい深さを加えない限り、ノイズ理論、パーティクルダイナミクス、数学的原理についての概念を繰り返さない
- **職人技を繰り返し強調**：哲学は、最終的なアルゴリズムが数え切れない時間をかけて開発され、丁寧に洗練され、その分野の絶対的なトップからのものに見えるべきことを複数回強調する必要がある
- **創造的な余地を残す**：アルゴリズムの方向性について具体的にしながら、次のClaudeが極めて高いレベルの職人技で解釈的な実装選択を行う余地を残すほど簡潔に

哲学は次のバージョンを導き、アイデアを静的画像ではなくアルゴリズム的に表現させる必要があります。美しさはプロセスに宿り、最終フレームにはない。

### 哲学の例

**「Organic Turbulence」**
哲学：自然法則に制約されたカオス、無秩序から秩序が出現。
アルゴリズム的表現：多層Perlinノイズによって駆動されるフローフィールド。ベクトル力に従う何千ものパーティクル、その軌跡が有機的な密度マップに蓄積。複数のノイズオクターブが乱流領域と穏やかなゾーンを作成。色は速度と密度から生まれる - 速いパーティクルは明るく燃え、遅いものは影にフェード。

**「Quantum Harmonics」**
哲学：波のような干渉パターンを示す離散的実体。
アルゴリズム的表現：グリッド上に初期化されたパーティクル、それぞれがサイン波を通じて進化する位相値を持つ。パーティクルが近いとき、位相が干渉 - 建設的干渉が明るいノードを作成、破壊的干渉がボイドを作成。

## P5.JS実装

哲学と概念的フレームワークが確立されたら、コードで表現します。

### ⚠️ ステップ0：まずテンプレートを読む ⚠️

**重要：HTMLを書く前に：**

1. Readツールを使用して`templates/viewer.html`を**読む**
2. 正確な構造、スタイリング、Anthropicブランディングを**研究**
3. **そのファイルを文字通りの開始点として使用** - インスピレーションだけでなく
4. **固定セクションをそのまま維持**（ヘッダー、サイドバー構造、Anthropicの色/フォント、シードコントロール、アクションボタン）
5. **可変セクションのみを置換**（アルゴリズム、パラメータ、パラメータ用UIコントロール）

### 技術要件

**シード付きランダム性（Art Blocksパターン）**：
```javascript
// 再現性のために常にシードを使用
let seed = 12345;
randomSeed(seed);
noiseSeed(seed);
```

**パラメータ構造 - 哲学に従う**：
```javascript
let params = {
  seed: 12345,  // 再現性のために常にシードを含める
  // アルゴリズムを制御するパラメータを追加：
  // - 数量（いくつ？）
  // - スケール（どれくらい大きい？どれくらい速い？）
  // - 確率（どれくらいありそう？）
  // - 比率（どんな割合？）
};
```

### 職人技の要件

**重要**：マスタージェネレイティブアーティストによる無数のイテレーションを経て生まれたように感じるアルゴリズムを作成。すべてのパラメータを丁寧に調整。すべてのパターンが目的を持って出現することを確認。

- **バランス**：視覚的ノイズのない複雑さ、硬直性のない秩序
- **色の調和**：思慮深いパレット、ランダムなRGB値ではない
- **構成**：ランダム性の中でも、視覚的階層とフローを維持
- **パフォーマンス**：アニメーションの場合はリアルタイムに最適化
- **再現性**：同じシードは常に同一の出力を生成

### 出力形式

出力：
1. **アルゴリズム哲学** - ジェネレイティブ美学を説明するmarkdownまたはテキスト
2. **単一HTMLアーティファクト** - `templates/viewer.html`から構築された自己完結型インタラクティブジェネレイティブアート

## インタラクティブアーティファクトの作成

ジェネレイティブアートの探索を可能にするために、単一の自己完結型HTMLアーティファクトを作成。このアーティファクトがclaude.aiまたは任意のブラウザで即座に動作することを確認 - セットアップ不要。すべてをインラインで埋め込む。

### 重要：固定と可変の区別

**固定（そのまま含める）：**
- レイアウト構造（ヘッダー、サイドバー、メインキャンバスエリア）
- Anthropicブランディング（UI色、フォント、グラデーション）
- サイドバーのシードセクション
- サイドバーのアクションセクション

**可変（各アートワーク用にカスタマイズ）：**
- p5.jsアルゴリズム全体
- パラメータオブジェクト
- サイドバーのパラメータセクション
- 色セクション（オプション）

## リソース

このスキルには役立つテンプレートとドキュメントが含まれています：

- **templates/viewer.html**：すべてのHTMLアーティファクトの必須開始点
- **templates/generator_template.js**：p5.jsのベストプラクティスとコード構造原則のリファレンス
